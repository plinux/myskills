# ESU ECoS 协议

用于处理 ESU ECoS/ECoS2 控制器的 PC Interface：TCP 文本协议、对象模型、命令/事件解析、机车/道岔/反馈/编程流程和硬件版本兼容性判断。

## 资料选择

- 实现命令编码、回复解析、状态机或联调真实控制器时，先读 `references/protocol.md`。
- 问题涉及 `50200`、`50210`、`50220`、`commandstationtype`、`hardwareversion`、`protocolversion` 或功能差异时，读 `references/hardware-compatibility.md`。
- 需要核对官方文档、本地实现依据或引用来源时，读 `references/source-map.md`。

## 基本判断

- ECoS PC Interface 是 TCP 文本协议，默认端口 `15471`，命令以换行结束。
- 不要按 `50200`、`50210`、`50220` 硬编码协议分支；先连接基础对象 `1`，读取型号、协议版本、硬件版本、应用版本和功能标志。
- 以能力和返回码决定兼容逻辑；遇到未知选项、未知对象或旧固件缺少字段时降级，不要推断控制器不可用。
- `<REPLY ...>` 是请求响应，`<EVENT ...>` 是订阅事件；事件可能与请求响应交错，解析器必须按块处理。
- 修改机车、道岔、编程轨或电源状态前，真实设备场景必须确认用户授权；这些命令会直接影响轨道电源、车辆运行和 CV 写入。

## 输出要求

分析协议或代码时，输出应包含：

- 使用的协议版本/控制器信息，或说明无法取得。
- 具体命令、对象 ID、选项、返回块和错误码解释。
- 是否需要 `request(..., view)` 或 `request(..., control)`，以及释放时机。
- 对 `50200`、`50210`、`50220` 的判断依据：硬件型号差异、功能探测结果，或官方/实现来源。
- 对真实设备的风险说明，尤其是 `set(1, go|stop)`、机车速度/方向、附件切换和 CV 写入。
