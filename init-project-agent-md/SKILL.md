# 项目 Agent 文件初始化

在当前项目中初始化项目级智能体提示文件和配套文档骨架。适用于 Codex、Claude Code 和其他遵循项目提示文件的智能体。

## 输入

用户可以提供语言或技术栈参数，例如 `cpp`、`python`、`go`、`typescript`。没有参数时，从项目文件推断。

## 环境判断

1. 先识别当前项目已有文件：
   - `AGENTS.md`、`.agents.md`、`.codex/` 表示通用 Agent 或 Codex 规则面。
   - `CLAUDE.md`、`.claude/` 表示 Claude Code 规则面。
2. 再结合当前会话环境判断：
   - Claude Code 会话优先创建或更新 `CLAUDE.md`。
   - Codex 或其他 Agent 会话优先创建或更新 `AGENTS.md`。
3. 如果用户明确要求某个文件，按用户要求执行。
4. 如果项目同时需要多 Agent 使用，创建或更新两个文件，并保持公共规则一致、客户端专属规则分开。

## 执行流程

### 1. 项目检测

检查当前目录是否已有 `.git`、构建文件、源码目录、测试目录、README、已有提示文件和文档目录。

文档目录按优先级选择：`Docs/`、`docs/`、`DOCS/`。都不存在时创建 `docs/`。

### 2. 存量项目分析

对存量项目先读取现有规范，不直接覆盖：

- 目录结构和核心模块。
- 已有 `AGENTS.md`、`CLAUDE.md`、README 和文档。
- 代码命名、缩进、注释和错误处理风格。
- 构建系统和测试框架。
- 最近提交日志格式。

### 3. 新项目初始化

对新项目使用通用默认结构，但仍根据用户提供的语言或检测到的技术栈填充命令占位。

### 4. 创建或更新提示文件

目标文件是 `AGENTS.md`、`CLAUDE.md` 或两者。内容应包含：

- 项目概述和技术栈。
- 运行环境和依赖要求。
- 关键目录结构。
- 构建、测试、格式化和常用验证命令。
- 代码风格、命名、注释和错误处理规则。
- 文档更新要求。
- Git 提交和测试门禁要求。
- 当前智能体专属规则；客户端专属内容不得混入另一个客户端文件。

如果目标文件已存在，只合并缺失规则并保留已有用户内容；覆盖前必须确认。

### 5. 创建文档骨架

在选定文档目录下创建缺失文件：

- `Arch.md`：架构设计索引和设计文档规范。
- `HLD.md`：整体架构设计模板。
- `Test.md`：测试报告汇总。
- `Perf.md`：性能测试简报。
- `UT.md`：单元测试简报。
- `Cov.md`：覆盖率测试简报。

创建缺失目录：

- `benchmark/`
- `unittest/`
- `coverage/`

目录为空时添加 `.gitkeep`。

### 6. 填充模板

模板必须使用中文，日期使用 `YYYY-MM-DD`。存量项目要填入真实检测结果；无法确定的内容保留为明确占位，并在最终报告中列出需要用户确认的项。

基本模板字段：

- 设计文档：设计时间、基准 commit、状态、完成提交、模块职责、接口和非功能需求。
- 测试文档：测试时间、commit、结果简述、详细报告链接。
- 性能文档：测试机器、工作负载、核心指标、详细报告链接。
- 覆盖率文档：行覆盖率、函数覆盖率、分支覆盖率、详细报告链接。

## 安全边界

- 不修改源代码或测试代码。
- 不覆盖已有提示文件、README 或设计文档；先读取、合并，再对覆盖操作确认。
- 不把 Claude Code 专属说明写入通用 `AGENTS.md`，不把 Codex 专属说明写入只面向 Claude Code 的 `CLAUDE.md`。
- 不在项目外创建文件。

## 最终报告

完成后列出：

- 新建或更新的提示文件。
- 新建或更新的文档和目录。
- 检测到的技术栈、构建命令、测试命令。
- 保留的已有规范。
- 需要用户确认的占位内容。
