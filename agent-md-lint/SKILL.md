# Agent 提示文件整理

整理项目级智能体提示文件，使其精炼、无冲突、可执行。适用文件包括 `AGENTS.md`、`CLAUDE.md`、`.claude/CLAUDE.md` 和 `.claude/AGENTS.md`。

## 目标选择

1. 优先使用用户明确指定的文件。
2. 未指定时，按当前环境和项目文件判断：
   - Claude Code 或项目已有 `CLAUDE.md` / `.claude/CLAUDE.md`：优先处理 Claude 文件。
   - Codex 或通用 Agent 环境、项目已有 `AGENTS.md` / `.agents.md`：优先处理 `AGENTS.md`。
   - 用户要求整理所有配置文件时，只处理当前项目内的 `AGENTS.md`、`CLAUDE.md`、`.claude/CLAUDE.md`、`.claude/AGENTS.md`。
3. 若多个候选文件同等匹配且用户没有说明目标，先问一个简短问题确认。

## 检查项

### 重复规则

查找同一规则在多个章节、多个列表项或不同措辞中重复出现的情况，例如缩进、命名、测试命令、提交规则或工具偏好重复定义。

处理方式：保留最具体、作用域最清楚的一处，合并或删除其余重复内容。

### 上下文冲突

查找同一主题给出不同值或互相排斥的要求，例如缩进数量、命名风格、覆盖率目标、默认工具、提交格式或文档路径不一致。

处理方式：
- 用户最近明确要求的新规则优先。
- 更深层、作用域更具体的规则优先。
- 无法判断权威来源时，先向用户展示冲突点并确认，不擅自选择。

### 歧义和不可执行表述

查找缺少数值、作用域、触发条件或执行边界的描述，例如“合理超时”“文件不要太大”“按惯例处理”“必要时更新文档”。

处理方式：基于文件上下文和项目现有做法改成具体规则；无法推断时只提出候选方案并询问。

### 冗长内容

删除背景叙事、重复解释、过程回顾、不会影响执行的说明、过时示例和空章节。保留能约束 Agent 行为的规则。

## 执行流程

1. 读取目标文件完整内容。
2. 列出发现项，按重复、冲突、歧义、冗长分类。
3. 先处理无争议的重复和冗长内容。
4. 再处理冲突和歧义；不确定时先询问。
5. 应用改写，保持原有语义和用户意图。
6. 输出简短报告，说明合并、删除、澄清和保留的关键点。

## 输出标准

整理后的文件必须满足：

- 每条规则只在一个权威位置出现。
- 不存在同一主题的互相矛盾要求。
- 每条要求都有明确作用域、触发条件或执行方式。
- 章节顺序清晰，同级标题和列表风格一致。
- 删除不影响执行的废话，但不发明原文件没有的新规则。
- 不修改当前项目之外的文件。

## 禁止行为

- 不确定内容是否冗余时，不要静默删除。
- 不要把 Claude Code 专属规则写进通用 `AGENTS.md`，也不要把 Codex 专属规则写进只面向 Claude Code 的 `CLAUDE.md`。
- 不要递归改写目标文件引用的其他文档，除非用户明确要求。
